### 低代码平台生成表单 (CRUD)

下面通过创建一个 `开发客户` 记录表来说明一下 **云程** 低代码平台 (CRUD) 的能力，主要包括下面几个步骤。

1. 创建应用
2. 创建数据实体（数据库表）
3. 构建数据实体的创建表单（对应 CRUD 的增加动作）
4. 构建数据的查询表格（包含 CRUD 的查询、编辑、删除动作）

> 低代码开发一般有单纯的生成前端表单代码，以及像云程一样的商业系统，通过可视化方式完成前端后台需要的完整功能，前者只是减少了前端写页面的时间，仍然需要开发人员手动编写各种业务代码。

#### 5. 使用小结

##### 优点
1. 使用该平台，能够在不需要编码的情况下，实现简单数据的增删查改功能（单表维度，复杂关联未知）。

##### 不足或隐患
1. 使用过程本身需要较强的开发背景知识才能使用好，对于一般非技术的业务人员或产品经理可能不友好（涉及数据库建索引）。
2. 复杂数据查询可能较难定义，且可能不适合数据量大的业务（黑盒无法优化）。
3. 要求业务去适应平台的功能（平台提供什么功能就用什么功能，和平时通过代码开发的项目不同）。
4. 有图片等静态资源上传的业务，可能会不太友好，图片保存到部署应用的服务器，加载时会消耗大带宽。

简单独立的业务可以尝试使用，预期有复杂功能业务谨慎使用，且由于数据都是平台独立的，难以和已有的系统对接交互。

> 因为此前没有低代码及相关平台使用经验，上述观点仅是短期内体验得出，建议产品先尝试一下是否能够满足一些业务需求。
