### 1. 组织架构接口字段说明

接口说明：MDM 提供以下入参的接口，SFA 通过该接口操作企业微信通讯录，包括创建组织架构、删除组织架构、移动组织架构（修改
parentId 和 fullName）。

> SFA 只有在操作 `外企德科（用人单位）`、`佩琪（用人单位）`、`猎聘（用人单位）`、`嘉驰（用人单位）`、`58魔方（用人单位）`
> ，及其子架构下才会调用该接口，MDM 也应校验该接口的操作数据范围。

请求参数

| 字段名       | 是否必填 | 类型      | 字段说明                                              |
|-----------|------|---------|---------------------------------------------------|
| id        | 否    | integer | 该组织架构的企业微信 id, 如果 id 为空时，表示需要创建架构，需要在接口返回值中返回该 id |
| name      | 必填   | string  | 组织架构名称                                            |
| fullName  | 必填   | string  | 组织架构全称                                            |
| parentId  | 必填   | integer | 组织架构的父节点的企业微信 id                                  |
| isDeleted | 必填   | boolean | true 删除, false 未删除                                |

返回值

| 字段名 | 是否必填 | 类型      | 字段说明                                        |
|-----|------|---------|---------------------------------------------|
| id  | 否    | integer | 如果是创建新的组织架构，需要返回企业微信中的组织架构 id，SFA 保存到自身数据库中 |

### 2. 人员同步接口字段说明

接口说明：MDM 提供以下入参的接口，SFA 通过该接口修改用户在企业微信中的信息，需要修改的值包括：手机、姓名、部门、职位、对外信息企业信息(固定值: `海天外部合作伙伴`)，MDM 需要校验如果该记录为内部员工记录，停止调用企业微信修改信息接口。

| 字段名      | 是否必填 | 类型      | 字段说明                                              |
|----------|------|---------|---------------------------------------------------|
| id       | 必填   | integer | 该值为人员记录在 SFA 数据库中的主键                              |
| name     | 必填   | string  | 姓名，姓名规则：承包商名简称+姓名+重名时的编码，如 `德科张三1` 或 `德科李四`(没有重名) |
| realName | 必填   | string  | 真实姓名                                              |
| phone    | 必填   | string  | SFA 中的手机号                                         |
| idCard   | 必填   | string  | 身份证号                                              |
| deptId   | 必填   | integer | 用户所属企业微信中的组织架构 id                                 |
| postName | 否    | string  | 职位名称                                              |
| status   | 必填   | integer | 1 启用，2 停用，3 删除                                    |
