##### 注重代码的格式化

> 程序写出来是给人看的，附带能在机器上运行--From 《计算机程序的构造和解释》

不是只有你的穿着外表代表着你的形象，还有你写的代码。写每行一代码时考虑别人的阅读感受。

注重代码的格式化，最简单的来说就是，该加空格的地方加空格，该换行的地方换行。 具体格式化的规则，如果团队中没有特殊要求，保持 IDE 默认设置即可，不必重复发明轮子，也不需死记硬背。

下面是一个简短的例子。

```java
// 没有格式化，拥挤的代码
if(condition){
    ... 
}

// 有合适空格的代码
if (condition) {
    ...
}
```

请经常使用这个快捷键 `Ctrl + Alt + L` ，让机器为你自动完成这些。

> “如果你是个木匠，你要做一个漂亮的衣柜，你不会用胶合板做背板，虽然这一块是靠着墙的，没人会看见。你自己知道它就在那儿，所以你会用一块漂亮的木头去做背板。如果你想晚上睡得安稳的话，就要保证外观和质量都足够好。” --- From《乔布斯传》

##### DRY 原则（Don't repeat yourself）

不要重复自己，提倡代码重用。Copy & Paste 很方便
