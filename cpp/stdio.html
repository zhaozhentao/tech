第 7 章 输入与输出

标准输入/标准输出简介

前面学习的程序中，我们使用 printf 来向屏幕打印信息，其实就是在使用 stdout 标准输出。 计算机科学中三个极其重要的基础概念：标准输入（stdin）、标准输出（stdout） 和 标准错误（stderr）。

简单的说，一个进程，天然就需要有接受输入和产生输出的需求（反过来想一下，如果一个程序既不接受输入，也不会产生输出，相当于没有产生任何影响，那这个程序还有什么意义？）。

所以，就需要有一个约定，来规定进程应该从哪里获得输入，进程的输出应该输出到哪里，这个约定就是一个"标准"。

标准输入

是程序读取输入数据的默认通道，通常是用户的键盘。

标准输出

是程序输出正常结果和数据的默认通道，通常是终端屏幕（Console/Terminal）。

标准错误

是程序输出错误消息、警告和诊断信息的专用通道。和标准输出一样，默认也是终端屏幕。

接下来，我们将通过库函数来认识一下 标准输入 / 标准输出 / 标准错误。

标准输入

最简单的从 标准输入 获取数据的方式就是使用 getchar 函数，该函数每次从键盘（默认）中读取一个字符。 


							
								#include <stdio.h>

								int main() {
									// 从标准输入读取一个字符
									char c = getchar();

									printf("读取到的是 %c\n", c);

									return 0;
								}

认识 EOF

getchar 函数在每次被调用的时候返回下一个输入的字符，如果遇到文件结束时（到目前为止，我们的标准输入还是键盘，未涉及文件），返回 EOF（End Of File，在头文件 stdio.h 中定义，其值是 -1）。

下面的程序在启动后，我们按下 Ctrl + d（向标准输入发送一个 EOF），可以看到程序会输出 -1。EOF 常用来检测是否读完一个文件。 

							
								#include <stdio.h>

								int main() {
									// 从标准输入读取一个字符
									int c = getchar();

									printf("读取到的是 %d\n", c);

									return 0;
								}

改变标准输入（重定向）

既然说 标准输入 是一种约定，默认是从键盘获取，那么我们是否不从键盘中读取输入，而从其他地方去读取呢？答案是肯定的， 在许多情况下，可以使用符号 '<' 让一个文件代替键盘作为进程的标准输入。 

							
								#include <stdio.h>

								int main() {
										char c;
								
										while ((c = getchar()) != EOF)
												putchar(c);
								
										return 0;
								}

使用下面的命令编译并运行上面的程序，可以看到 main.c 文件中的内容被打印出来，这是因为此时文件 main.c 作为进程的标准输入， 文件中的所有内容被 putchar 一个个字符输出到屏幕，直到文件结束（EOF）。 

							
								$ ./a.out < main.c

除了使用 '<' 符号来实现标准输入的重定向，还可以使用 管道 机制来实现。继续使用前面的程序，运行下面的命令， 也可以看到相同的效果。第一个命令 'cat main.c' 的输出通过 管道 符号 '|' 成为了第二个进程的标准输入。 

							
								$ cat main.c | ./a.out

通过这两种方式，在不改变一行源代码的情况下，就完成了标准输入的重定向，可见 Unix/Linux 的强大和设计之美。

标准输出

格式化输出 printf 函数

前面的学习中，我们已经多次使用 printf 函数去打印一些信息，这其实就已经是在使用 stdout 标准输出 了，正如前面说的， 标准输出默认的情况下就是输出到终端屏幕中。 

							
								#include <stdio.h>

								int main() {
										printf("hello world\n");
								
										return 0;
								}

printf 函数的完整的描述为： 

							
								int printf(char *format, arg1, arg2, ...)

其中 format 参数包含普通字符以及待替换为实际数据的 占位符 ，前面的学习已经多次使用了。 更多的占位符含义在下一页。

标准输出的重定向

与标准输入可以重定向一样，标准输出也可以重定向，使程序打印的内容，输出其他地方。。 

							
								#include <stdio.h>

								int main() {
										printf("hello world\n");
								
										return 0;
								}

编译上面的程序，并按下面的命令执行程序，就可以看到原本输出到屏幕的 hello world 被输出到了 output 文件中。'>' 符是标准输出重定向的方法之一。 

							
								$ ./a.out > output

使用管道符 '|' 也可以实现标准输出的重定向，下面的命令把 hello world 输出到后面一个命令 wc -l 中，该命令会统计文本的行数。 

							
								$ ./a.out | wc -l
