#### 指针与数组

C语言中，指针与数组的关系十分密切，通过数组下标所能完成的操作都能使用指针完成，一般来说，用指针编写的程序比用数组下标编写的程序执行速度快，但通过数组下标编写的程序要容易理解一些。

##### 数组元素取址

回忆一下一些有关数组的操作。

```clike
// 数组的定义
int a[10];

// 从数组中取某一个元素
a[i];
```

假如有一个指向 int 类型的指针 p，那么就可以使用这个指针来完成数组下标的操作。

```clike
// 指向 int 类型的指针
int *p;

// 指向数组中的第 i 个元素(先找到第 i 个元素，然后取出该元素的地址)
p = &a[i];

// 读取或写入
int b = *p;

// 写入 1
*p = 1;
```

##### 数组名与指针

另外，C语言中数组名字代表数组的起始地址（例如上面的数组名 a ，就代表 &a[0]），所以有下面的关系

```clike
int *p;

// 通过数组名取出数组起始地址
p = a;

// 与上面的结果相同
p = &a[0];
```

注意，数组名 a，不是一个变量，不能像修改指针 p 一样，对 a 进行修改，比如 a = p。

##### 指针的运算

类似与数组通过下标取不同的元素，指针也有相应的操作。

```clike
int a[10];

// 读取第 5 号元素
int c = a[5];

int *p = a;

// 读取第 5 号元素
int d = *(p + 5);
```

> 提示，可以总结出这样的结论，p + xx (xx 是一个整数) 等于 p 指向的地址加上 xx 乘以 p 指向的类型大小，例如 p 是指向 int 的指针，p + 2 就等于 p + 2 * 4 (int 的大小为 4)。

#### 字符指针与函数

字符串常量是一个字符数组，例如:

```clike
"I am a string"
```

字符串总是以 '\0' 结尾，所以程序可以通过检查空字符找到字符串的结尾，字符串常量占据的存储单元数比双引号内的字符数大 1 ，比如上面的字符串，实际上长度是 14。

通过下面的字符数组，我们同样可以打印出 `Hello World`。

```cilke
#include <stdio.h>

int main() {
        char a[13] = {'H', 'e', 'l', 'l', 'o', ' ', 'W', 'o', 'r', 'l', 'd', '\n'};

        printf(a);

        return 0;
}
```

字符串常量最常见的用法是作为函数参数，例如

```clike
printf("hello world\n");
```

当一个字符串出现在程序中时，实际上是通过字符指针访问该字符串的。从前面两个示例可以看出，不论是 printf("Hello world\n") 还是 printf(a); 都能实现打印.
